### Step 8: 自動フェーズ実行

承認を得たら、`01-phases/` の各フェーズを順番に実行：

1. **Phase 1**: アイデア評価（`idea_input.md` を基に評価・選定）
2. **Phase 2**: アプリ名決定（39言語ARB生成）
3. **Phase 3**: 競合深掘り調査
4. **Phase 4**: ビジネス設計
5. **Phase 5**: 技術設計
6. **Phase 6**: 要件定義（ハイブリッド方式）
7. **Phase 7**: 仕様書作成
8. **Phase 8**: リポジトリ構築（`make clone-repo`）

各Phaseの詳細は `01-phases/phase{N}-*.md` を読み込んで実行。

**重要**: Phase 1 では `docs/project/idea_input.md` の内容を入力として使用し、
ゼロからアイデアを生成するのではなく、ユーザーのアイデアを基に評価・最適化を行う。

#### Phase 8 完了後（必須）: 未コミット変更の同期

`make clone-repo` はリモートのコミット済みテンプレートをクローンするため、
flutter-template のローカル未コミット変更は含まれない。**以下を必ず追加実行すること。**

```bash
# flutter-template 作業ツリーを新リポジトリへ同期（未コミット変更を含む）
rsync -av \
  --exclude='.git/' \
  --exclude='build/' \
  --exclude='.dart_tool/' \
  --exclude='.flutter-plugins' \
  --exclude='.flutter-plugins-dependencies' \
  . ../{REPO_NAME}/

# 変更をコミット・プッシュ
cd ../{REPO_NAME}
git add .
git commit -m "sync: flutter-templateの未コミット変更を反映" || echo "変更なし"
git push
```

**除外ルール**:
- `build/`, `.dart_tool/`: ローカルビルドキャッシュは不要
- `docs/project/` は **除外しない**（Phase 1-7 の成果物が格納されているため、同期対象に含める）
