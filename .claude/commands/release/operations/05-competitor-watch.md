# 👁️ Operations Phase 5: 競合監視

## 目的
競合の動向を継続的に監視し、市場変化に迅速に対応する

---

## 前提条件
- Phase 3で競合分析が完了済み
- 主要競合3-5社が特定済み
- `docs/project/competitors.md` に競合リストが存在

---

## 監視対象と頻度

| 監視項目 | 頻度 | 所要時間 | 目的 |
|---------|------|---------|------|
| アプリ更新 | 週1回 | 15分 | 新機能・改善の把握 |
| 価格変更 | 週1回 | 5分 | 価格戦略の把握 |
| ASO変更 | 月1回 | 30分 | マーケティング戦略 |
| レビュー動向 | 月1回 | 30分 | ユーザー満足度 |
| 市場参入者 | 月1回 | 15分 | 新規競合の発見 |
| 総合分析 | 四半期 | 2時間 | 戦略見直し |

## 🤖 Claude Code への指示

**トークン効率化:**
- 競合情報は差分のみ記録
- 変更なしの週はスキップ可能
- プロファイルは重要項目のみ更新

**実行モード:**
1. `--weekly`: 週次更新チェック
2. `--monthly`: 月次レポート生成
3. `--quarterly`: 四半期分析
4. `--new-competitor [名前]`: 新規競合プロファイル作成

**データソース:**
- App Store Connect / Google Play Console API
- WebSearch による最新情報収集
- レビュー分析ツール連携（オプション）
