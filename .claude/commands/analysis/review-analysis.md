**App Store / Google Play のレビューを分析し、評価向上施策を提案**

## 🎯 概要

このコマンドは、ストアレビューを自動分析し、以下を実行します:
- 最新レビューの感情分析（ポジティブ/ネガティブ）
- 頻出する問題点・要望の抽出
- 評価別（★1-5）のトレンド分析
- 競合アプリとの比較
- レビュー返信テンプレート生成
- 評価向上のための優先施策提案

**IMPORTANT:** このコマンド実行前に、必ず以下を参照してください:
- **[トークン効率化ベストプラクティス](../../docs/best-practices/token-efficiency.md)** - 10個の黄金ルール

### 6. 評価向上施策

#### 6.1 レビュー依頼の最適化

```bash
「アプリ内レビュー依頼（In-App Review）の最適化:

🎯 最適なタイミング:

✅ Good Timing:
   1. ポジティブ体験直後:
      - 主要機能の成功利用後
      - 目標達成時（タスク完了等）
      - 「神アプリ」と感じた瞬間

   2. 定着後:
      - 7日間連続利用
      - 10回以上の起動
      - 有料機能の購入後

   3. サポート対応成功後:
      - 問い合わせへの満足度が高い場合

❌ Bad Timing:
   1. ネガティブ体験時:
      - エラー・クラッシュ直後
      - 広告表示直後
      - 課金失敗時

   2. アプリ起動直後:
      - 初回起動時（まだ価値を感じていない）
      - インストール後 1分以内

   3. 過剰な頻度:
      - 前回依頼から 30日以内
      - 同じセッション内で複数回

📱 実装コード例（Flutter）:
   ```dart
   // InAppReview パッケージ使用
   final InAppReview inAppReview = InAppReview.instance;

   Future<void> requestReview() async {
     // 条件チェック
     if (await shouldRequestReview()) {
       if (await inAppReview.isAvailable()) {
         inAppReview.requestReview();
       }
     }
   }

   Future<bool> shouldRequestReview() async {
     // 7日間連続利用 & 10回以上起動 & 30日以内に依頼なし
     final days = await consecutiveDays();
     final launches = await totalLaunches();
     final lastRequest = await lastReviewRequestDate();

     return days >= 7 &&
            launches >= 10 &&
            (lastRequest == null ||
             DateTime.now().difference(lastRequest).inDays > 30);
   }
   ```

💡 コンバージョン率向上:
   - 最適タイミング: CVR 20-30%
   - 不適切タイミング: CVR 5-10%
   - 差: 3-6倍」
```
