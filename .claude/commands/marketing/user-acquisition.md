# 📈 ユーザー獲得戦略コマンド

<!-- LANGUAGE: ja -->
<!-- RESPONSE_LANGUAGE: Japanese -->

**リリース後のユーザー獲得戦略を自動分析・提案**

## 🎯 概要

このコマンドは、リリース後のアプリに対して以下を実行します:
- 現在のダウンロード数・インストール状況の分析
- ターゲットユーザー層の再評価
- 効果的なユーザー獲得チャネルの提案
- 広告予算配分の最適化提案
- オーガニックグロース戦略の立案

**IMPORTANT:** このコマンド実行前に、必ず以下を参照してください:
- **[トークン効率化ベストプラクティス](../../docs/best-practices/token-efficiency.md)** - 10個の黄金ルール

---

## 📋 実行内容

### 1. 現状分析（Firebase Analytics データ活用）

```bash
「Firebase Analytics から以下のデータを取得します:

📊 基本指標:
   - DAU (Daily Active Users)
   - MAU (Monthly Active Users)
   - インストール数
   - アンインストール率
   - セッション時間
   - リテンション率（1日後、7日後、30日後）

🌍 ユーザー属性:
   - 国/地域別分布
   - デバイス（iOS）比率
   - OS バージョン分布
   - 言語設定分布
   - 初回起動からの経過日数

📱 行動データ:
   - スクリーン滞在時間
   - 主要機能の利用率
   - コンバージョン率（課金、広告クリック等）
   - ユーザーフロー分析」
```

### 2. ターゲット市場の再評価

```bash
「現在のデータから最適なターゲット市場を特定します:

🎯 高パフォーマンス市場:
   - リテンション率が高い国/地域
   - ARPU（ユーザー平均収益）が高いセグメント
   - オーガニック成長率が高い地域
   - 競合が少ない市場

🚫 低パフォーマンス市場:
   - アンインストール率が高い地域
   - セッション時間が短いセグメント
   - 広告予算対効果が低い市場

💡 推奨アクション:
   - 高パフォーマンス市場への集中投資
   - 低パフォーマンス市場からの撤退検討
   - 新規市場の開拓可能性分析」
```

### 3. ユーザー獲得チャネル戦略

#### 3.1 オーガニックチャネル（無料）

```bash
「コストをかけずにユーザーを獲得する戦略:

📱 ASO（App Store Optimization）:
   - キーワード最適化
   - スクリーンショット A/B テスト
   - アプリ説明文の改善
   - レビュー評価の向上施策

🌐 SNS マーケティング:
   - Twitter/X でのバズ施策
   - Reddit コミュニティ投稿
   - Product Hunt への掲載
   - Indie Hackers での紹介

📝 コンテンツマーケティング:
   - ブログ記事作成
   - チュートリアル動画
   - プレスリリース配信
   - Medium/Qiita での技術記事

🔗 バイラル機能:
   - 友達招待プログラム
   - ソーシャルシェア機能
   - リファラルボーナス
   - 口コミ促進施策」
```

#### 3.2 ペイドチャネル（有料広告）

```bash
「広告予算を効果的に配分する戦略:

📊 Apple Search Ads:
   - 推奨予算: 初期 $10-20/日
   - ターゲット: ブランドキーワード + 競合キーワード
   - CPI（インストール単価）目標: $0.50-2.00
   - ROAS（広告費用対効果）目標: 150%以上

🎯 Google App Campaign:
   - 推奨予算: 初期 $10-20/日
   - ターゲット: リテンション重視ユーザー
   - CPI 目標: $0.30-1.50
   - 最適化指標: ROAS または LTV

📱 SNS 広告（Facebook/Instagram/TikTok）:
   - 推奨予算: 初期 $5-10/日
   - ターゲット: ペルソナに基づくオーディエンス
   - CPI 目標: $0.50-2.00
   - クリエイティブ A/B テスト必須

🌍 Reddit Ads（ニッチ市場向け）:
   - 推奨予算: 初期 $5/日
   - ターゲット: 関連サブレディット
   - CPI 目標: $0.30-1.00
   - コミュニティエンゲージメント重視

💡 予算配分最適化:
   - 初期: 各チャネル均等配分でテスト
   - 2週間後: CPI/ROAS データに基づき再配分
   - 1ヶ月後: 高パフォーマンスチャネルに集中
   - 継続的な A/B テスト実施」
```

### 4. 成長ハック施策

```bash
「急成長を実現する非従来型戦略:

🚀 プロダクトハント戦略:
   - 最適な投稿タイミング（火-木曜日、PST 0:01）
   - Maker コミュニティでの事前宣伝
   - ハンター（影響力のある投稿者）への依頼
   - 目標: 上位5位入り → 500-2000 インストール

📱 App Store フィーチャー獲得:
   - Apple に直接提出
   - 季節・イベントに合わせた機能追加
   - デザイン品質の向上
   - 目標: 地域別フィーチャー → 数千インストール

🎁 ローンチキャンペーン:
   - 期間限定プロモーション（初月無料等）
   - インフルエンサーへのプロモコード提供
   - 早期アダプターへの特典
   - 目標: 初週 500-1000 インストール

🌐 コミュニティ戦略:
   - Discord/Slack コミュニティ構築
   - ユーザー主導のフィードバックループ
   - ベータテスターコミュニティ
   - アンバサダープログラム」
```

### 6. KPI トラッキング

```bash
「追跡すべき重要指標:

📊 獲得指標:
   - インストール数（オーガニック vs ペイド）
   - CPI（インストール単価）
   - CVR（ストアページ訪問→インストール率）
   - チャネル別パフォーマンス

💰 収益指標:
   - ARPU（ユーザー平均収益）
   - LTV（Life Time Value）
   - ROAS（広告費用対効果）
   - CAC（顧客獲得コスト）

🔄 リテンション指標:
   - 1日後リテンション率
   - 7日後リテンション率
   - 30日後リテンション率
   - Churn Rate（解約率）

📈 成長指標:
   - DAU/MAU 比率（粘着性）
   - バイラル係数（K-factor）
   - オーガニック成長率
   - ネットプロモータースコア（NPS）」
```

## 🤖 Claude Code への実行指示

以下の手順で自動実行します:

### Step 1: Firebase Analytics データ取得

```bash
「Firebase Console から Analytics データを取得します:

1. プロジェクト選択（dev/prod）
2. 以下の期間でデータ抽出:
   - 直近 7 日間（短期トレンド）
   - 直近 30 日間（中期トレンド）
   - リリースから現在まで（全期間）

3. 重要指標のエクスポート:
   - ユーザー獲得レポート
   - リテンションレポート
   - オーディエンスレポート
   - イベント分析レポート」
```

### Step 2: 競合アプリ調査

```bash
「WebSearch で競合の最新動向を調査:

🔍 調査対象:
   - 同カテゴリ上位 5 アプリ
   - App Store/Google Play ランキング推移
   - 最近のマーケティング施策
   - レビュー評価の変化

💡 分析ポイント:
   - 成功しているチャネル
   - クリエイティブの傾向
   - プロモーション戦略
   - 差別化ポイント」
```

### Step 3: 戦略レポート生成

```bash
「以下の構成でマークダウンレポートを生成:

📄 レポート構成:
   1. エグゼクティブサマリー
   2. 現状分析（データ可視化）
   3. ターゲット市場の再評価
   4. チャネル別戦略（優先順位付き）
   5. 予算別実行プラン
   6. KPI ダッシュボード設計
   7. 90日間実行ロードマップ
   8. リスク分析と対策

💾 保存先:
   docs/project/user_acquisition_strategy.md」
```

```bash
「戦略を実行可能なタスクに変換し、Dart に登録:

📋 タスク例:
   - ASO キーワード最適化
   - Product Hunt 掲載準備
   - Apple Search Ads キャンペーン設定
   - レビュー依頼メール送信
```
