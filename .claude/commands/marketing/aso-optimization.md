**App Store Optimization（ASO）を自動分析・最適化提案**

## 🎯 概要

このコマンドは、App Store / Google Play での検索順位向上のため、以下を実行します:
- 現在のASO状態の診断
- キーワード最適化（検索ボリューム×競合度分析）
- アプリタイトル・説明文の最適化提案
- スクリーンショット・プレビュー動画の改善案
- 競合アプリのASO戦略分析
- A/Bテスト計画の策定

**IMPORTANT:** このコマンド実行前に、必ず以下を参照してください:
- **[トークン効率化ベストプラクティス](../../docs/best-practices/token-efficiency.md)** - 10個の黄金ルール

## 📋 実行内容

### 1. ASO 健康診断

#### 1.1 現状スコアリング

```bash
「アプリの ASO 成熟度を 100点満点で評価:

📊 評価項目（各 10点満点）:

1. アプリタイトル:
   □ ブランド名 + 主要キーワード含有（5点）
   □ 30文字以内の最適長（App Store）/ 50文字（Google Play）（3点）
   □ 差別化ワード含有（2点）

2. サブタイトル（App Store）/ 簡潔な説明（Google Play）:
   □ 価値提案が明確（5点）
   □ 2-3個のキーワード含有（3点）
   □ 行動喚起含有（2点）

3. アプリ説明文:
   □ 最初の 3行で価値を訴求（3点）
   □ キーワード密度 3-5%（3点）
   □ 箇条書き・絵文字で読みやすい（2点）
   □ ユーザーレビュー引用（2点）

4. キーワードフィールド（App Store）:
   □ 100文字を完全活用（3点）
   □ 高ボリューム×低競合キーワード（5点）
   □ ロングテールキーワード含有（2点）

5. スクリーンショット:
   □ 主要機能を 3枚目までに（3点）
   □ テキストオーバーレイで価値訴求（3点）
   □ デバイス別最適化（2点）
   □ A/B テスト実施済み（2点）

6. プレビュー動画:
   □ 15-30秒の最適長（3点）
   □ 最初の 3秒でフック（5点）
   □ 音声なしでも理解可能（2点）

7. アプリアイコン:
   □ シンプルで記憶に残る（5点）
   □ カテゴリ内で差別化（3点）
   □ 小サイズでも視認可能（2点）

8. レビュー評価:
   □ ★4.5 以上（5点）
   □ レビュー数 1000+ （3点）
   □ 直近30日で新鮮なレビュー（2点）

9. アップデート頻度:
   □ 月1回以上（5点）
   □ リリースノート充実（3点）
   □ ユーザー要望への対応（2点）

10. ローカライゼーション:
    □ 主要5言語対応（5点）
    □ 各言語でキーワード最適化（3点）
    □ 現地のトレンド反映（2点）

💯 総合スコア:
   - 90-100点: 優秀
   - 70-89点: 良好（改善余地あり）
   - 50-69点: 要改善
   - 50点未満: 緊急対応必要」
```

### 2. キーワードリサーチ

#### 2.1 キーワード発見

```bash
「効果的なキーワードを発見する手法:

🔍 発見方法:

1. 競合アプリ分析:
   - 競合トップ5のキーワードを抽出
   - 自社にないキーワードをリストアップ
   - 競合が見落としているニッチキーワード

2. サジェスト機能:
   - App Store 検索窓のオートコンプリート
   - Google 検索のサジェスト
   - 「[メインキーワード] + ...」の組み合わせ

3. ユーザーレビュー分析:
   - 自社アプリのレビュー頻出語
   - 競合アプリのレビュー頻出語
   - ユーザーの言葉遣いを反映

4. トレンドキーワード:
   - Google Trends での季節変動
   - SNS でのバズワード
   - 時事ネタ・イベント関連

5. ロングテールキーワード:
   - 「[悩み] 解決」
   - 「[状況] おすすめ」
   - 「無料 [機能]」

💡 キーワード例（ツール系アプリの場合）:
   - メインキーワード: 「タスク管理」
   - 関連キーワード: 「TODO」「リスト」「メモ」
   - ロングテール: 「タスク管理 シンプル」「TODO アプリ 無料」
   - 競合キーワード: 「[競合名] 代替」「[競合名] より軽い」」
```

#### 2.2 キーワード評価

```bash
「各キーワードを 検索ボリューム × 難易度 で評価:

📊 評価指標:

1. 検索ボリューム（月間検索数）:
   - 高（10,000+）: 5点
   - 中（1,000-10,000）: 3点
   - 低（100-1,000）: 2点
   - 極小（100未満）: 1点

2. 競合難易度:
   - 低（上位10アプリの平均評価 < 4.0）: 5点
   - 中（4.0-4.5）: 3点
   - 高（4.5+）: 1点

3. 関連性:
   - 高（アプリの主要機能）: 5点
   - 中（付随機能）: 3点
   - 低（ユーザーの一部のみ関心）: 1点

💯 総合スコア = (ボリューム × 0.4) + (難易度 × 0.4) + (関連性 × 0.2)

📋 優先順位付け:
   1. Quick Wins（高ボリューム × 低競合）
   2. 戦略的（高ボリューム × 高競合、長期戦）
   3. ロングテール（低ボリューム × 低競合、ニッチ）
   4. 避けるべき（低ボリューム × 高競合）

💡 キーワード選定例:
   【Quick Wins】
   - 「タスク管理 シンプル」（ボリューム: 中、競合: 低、関連: 高）→ 3.8点

   【戦略的】
   - 「TODO アプリ」（ボリューム: 高、競合: 高、関連: 高）→ 3.4点

   【ロングテール】
   - 「買い物リスト 共有」（ボリューム: 低、競合: 低、関連: 中）→ 2.6点」
```

### 3. メタデータ最適化

#### 3.1 アプリタイトル最適化

```bash
「App Store / Google Play のタイトル最適化:

📱 App Store タイトル（30文字制限）:

構成パターン:
   [ブランド名] - [主要キーワード] & [差別化ワード]

✅ Good例:
   - 「TaskMaster - シンプルTODO」（20文字）
   - 「QuickMemo - 瞬速メモアプリ」（19文字）
   - 「FitTrack - 習慣化サポート」（18文字）

❌ Bad例:
   - 「TaskMaster」（キーワードなし）
   - 「最高のタスク管理アプリ TaskMaster Pro」（長すぎる、誇張表現）
   - 「TaskMaster - TODO,リスト,メモ,予定」（キーワード詰め込み）

🤖 Google Play タイトル（50文字制限）:

構成パターン:
   [ブランド名] - [主要キーワード1] & [主要キーワード2] | [差別化]

✅ Good例:
   - 「TaskMaster - TODOリスト & タスク管理 | シンプル無料」（32文字）
   - 「QuickMemo - メモ帳・ノート・リスト | 広告なし」（30文字）

💡 最適化のポイント:
   - ブランド名は短く（10文字以内推奨）
   - 主要キーワード 1-2個
   - 差別化ワード（「シンプル」「無料」「軽い」等）
   - 記号は「-」「&」「|」を適度に使用
   - 全角/半角を使い分けて文字数調整」
```

### 4. ビジュアル最適化

#### 4.1 スクリーンショット戦略

```bash
「コンバージョン率を最大化するスクリーンショット:

📸 構成（App Store: 最大10枚、Google Play: 最大8枚）:

【推奨構成】
1枚目: ヒーローショット（最も魅力的な画面 + キャッチコピー）
2枚目: 主要機能①（価値提案 + 画面）
3枚目: 主要機能②（差別化ポイント）
4枚目: 主要機能③
5枚目: ユーザーの声・レビュー
6-8枚目: その他機能・詳細

🎨 デザインのベストプラクティス:

1. テキストオーバーレイ:
   □ 大きく読みやすいフォント（40pt以上）
   □ 画面の上部30%にキャッチコピー
   □ コントラスト確保（白文字に黒影等）
   □ 3-7ワードの短いメッセージ

2. 実機画面:
   □ 実際のアプリ画面を使用
   □ 高解像度（ぼやけない）
   □ デバイスモックアップで囲む（オプション）
   □ 明るく鮮やかな配色

3. 配色:
   □ ブランドカラーで統一感
   □ 背景は明るい色（視認性）
   □ アクセントカラーで注目誘導

✅ Good例（1枚目ヒーローショット）:
```
【画面】
上部: 「3秒でタスク整理」（大きく太字）
中央: アプリのメイン画面（TODOリスト）
下部: 「シンプル・無料・広告なし」（小さめ）
```

❌ Bad例:
- テキストなし（何のアプリか不明）
- 文字が小さすぎる（読めない）
- 画面が複雑すぎる（理解できない）
- 暗すぎる（魅力的でない）

📊 A/Bテスト:
   - Apple Search Ads でスクリーンショット差し替えテスト
   - Google Play Console の実験機能活用
   - CVR 向上を継続的に測定」
```

### 5. ローカライゼーション

```bash
「多言語対応で世界市場を攻略:

🌍 優先言語（ダウンロード数順）:

1. 英語（米国）: 最優先（市場規模最大）
2. 中国語（簡体字）: 超巨大市場
3. 日本語: 高ARPU市場
4. スペイン語: 人口多数
5. ポルトガル語（ブラジル）: 成長市場
6. フランス語: 高所得市場
7. ドイツ語: 高課金率
8. 韓国語: 高ARPU市場

📝 ローカライゼーション戦略:

1. 機械翻訳 ✕ → ネイティブレビュー ○
   - DeepL / Google翻訳で下訳
   - ネイティブスピーカーが必ず確認
   - 文化的配慮（タブー回避）

2. キーワードの現地化:
   - 各言語で独自のキーワードリサーチ
   - 英語の直訳ではなく、現地での検索語

例: 「タスク管理」
   - 英語: task management, to-do list, productivity
   - 中国語: 任务管理, 待办事项, 效率
   - スペイン語: gestión de tareas, lista de tareas
   - ドイツ語: Aufgabenverwaltung, To-Do-Liste

3. ビジュアルの現地化:
   - 言語別スクリーンショット作成
   - 現地の文化に合わせたデザイン
   - 読み方向の調整（アラビア語等）

4. 段階的ローカライゼーション:
```

```bash
   - パターンB: ベネフィット型
```
