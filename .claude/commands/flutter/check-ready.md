# check-ready

## 概要
実装完了後のコード品質チェックを一括実行するコマンドです。フォーマット、静的解析、テストを順次実行し、プッシュ前の品質を保証します。

## 実行内容
1. **フォーマット**: `fvm dart format lib/ test/`
2. **静的解析**: `fvm flutter analyze`
3. **テスト実行**: `fvm flutter test --dart-define-from-file=dart_env/test.env`

## 使用タイミング
- **実装完了後（必須）**: 新機能や修正の実装が完了した時
- **プッシュ前（推奨）**: `git push` する前の最終チェック
- **PR作成前**: プルリクエストを作成する前
- **レビュー指摘対応後**: コードレビューの指摘を修正した後

## 実行方法
```bash
make check-ready
```

## エラー対処
- **フォーマットエラー**: 自動修正されるため再実行
- **静的解析エラー**: エラーメッセージに従って修正後再実行
- **テストエラー**: 失敗したテストを修正後再実行

## 関連コマンド
- `make format`: フォーマットのみ実行
- `make analyze`: 静的解析のみ実行
- `make test`: テストのみ実行
- `make push`: check-ready + git push を実行

## 注意事項
- すべてのチェックが成功するまでプッシュは控えてください
- Lefthookにより、プッシュ時に同様のチェックが自動実行されます
- 緊急時は `LEFTHOOK=0 git push` でスキップ可能ですが非推奨です
