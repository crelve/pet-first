# Pet First 機能仕様

## P0: 症状別ガイド

### 症状カテゴリ（犬/猫別）

| カテゴリ | 主要症状 |
|---------|---------|
| 嘔吐・下痢 | 急性嘔吐、血便、白色便、黒色便、慢性下痢 |
| 誤飲・中毒 | チョコレート、玉ねぎ、ぶどう、人間の薬、植物 |
| 痙攣・意識障害 | てんかん発作、震え、ふらつき、虚脱 |
| 外傷・出血 | 切り傷、爪折れ、咬傷、転落、骨折疑い |
| 呼吸器症状 | 呼吸困難、咳、くしゃみ、鼻血 |
| 皮膚・耳のトラブル | 痒み、脱毛、耳垢、皮膚炎、ホットスポット |
| 眼の症状（猫） | 目やに、第三眼瞼の露出、瞬膜 |
| 排尿障害 | 頻尿、血尿、出ない |

合計: 30+症状

## P0: 緊急度判定システム

### 判定ロジック

各症状に対して、付随条件で緊急度を変動させる：

```
Symptom: 嘔吐
- 単発で元気あり → 緑（自宅様子見）
- 反復、ぐったり → 赤（即病院）
- 血が混じる → 赤（即病院）
- 24時間続く → 黄（朝一受診）
```

UIでは「症状をタップ→チェックリスト→最終判定」の流れ。

### 緊急度別UI

| レベル | 背景色 | アイコン | 行動指示 |
|--------|-------|---------|---------|
| 赤 | 赤背景 | 警告サイレン | "今すぐ動物病院へ" + 病院検索ボタン |
| 黄 | 黄背景 | 注意マーク | "様子見、悪化したら受診" |
| 緑 | 緑背景 | チェックマーク | "自宅で対処OK、ステップ参照" |

## P0: 応急処置ステップ

### ステップ表示形式

各症状ごとに以下を表示：

1. **すぐにやること**: 番号付きステップ（最大5ステップ）
2. **やってはいけないこと**: 赤字で警告（NG行為）
3. **病院に連れて行くサイン**: チェックリスト
4. **持っていくべきもの**: 嘔吐物の写真等

### 例: 「犬の誤飲（チョコレート）」

```
🟥 即病院

【今すぐ】
1. 何をどれくらい食べたか確認
2. パッケージや残骸を回収
3. 動物病院に電話して状況説明
4. 病院に向かう

【NG】
✗ 自分で吐かせようとしない（一部の毒物は嘔吐で悪化）
✗ 様子を見ない（チョコは数時間で症状進行）
✗ 牛乳を飲ませない（脂肪分が吸収を促進）

【病院に持っていく】
- 食べたチョコのパッケージ
- 食べた量のメモ
- 食べた時間
```

## P0: オフライン対応

- 起動時にHive初期化
- 全症状情報をローカル保持
- 画像はアセットバンドル
- ネットワーク不要で全P0機能動作

## P0: ペイウォール

### 表示条件
- オンボーディング完了直後（ソフト訴求、スキップ可）
- 3症状目閲覧時（ハード、ペイウォールパス必須）

## P0: 広告

| 配置 | 形式 | 表示条件 |
|------|------|---------|
| Home下部 | バナー | フリープラン時のみ |
| SymptomList → SymptomDetail遷移時 | インター | 緊急度=緑のときのみ、フリーのみ |
| 詳細画面下部 | バナー | フリーのみ |

赤・黄レベル時は広告完全非表示（離脱防止+UX配慮）

## P0: 通知（M3）

| タイミング | 内容 |
|-----------|------|
| 月初 | "今月のペット健康チェック。元気ですか？" |
| 季節変わり目 | "夏の熱中症対策、今のうちに確認" |
| 初回起動24時間後 | "症状ガイドを試してみませんか" |

## P0: オンボーディング（M4）

3ステップ：
1. "ペットの異変、3秒で緊急度判定" — アプリのコア価値
2. "オフラインでも100%動作" — 信頼性
3. "犬・猫対応、30+症状" — カバー範囲

## P1: ペットプロフィール

- 名前、種類、品種、年齢、体重
- アレルギー、既往歴、服用薬
- 獣医に見せる用のサマリー画面
