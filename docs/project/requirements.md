# Pet First 要件定義

## Part 1: ステークホルダー要件

### プライマリユーザー
- 20-50代のペットオーナー（犬または猫を飼育中）
- 深夜・早朝にペットの体調不良で不安になる人
- 獣医に電話する前に「これって緊急？」と確認したい人

### 主要ユースケース

| ID | ユースケース | 優先度 |
|----|-------------|--------|
| UC-01 | ペットの症状を選んで応急処置を確認 | P0 |
| UC-02 | 緊急度を即座に判定（病院に行くべきか） | P0 |
| UC-03 | NG行為（やってはいけないこと）を確認 | P0 |
| UC-04 | オフラインでも全機能を使う | P0 |
| UC-05 | ペットプロフィールを記録 | P1 |
| UC-06 | 最寄りの動物病院を検索 | P1 |

## Part 2: EARS要件

### 通常要件

- ユーザーがアプリを起動したとき、システムは犬/猫タブと症状カテゴリリストを表示する
- ユーザーが症状を選択したとき、システムは緊急度（赤/黄/緑）と応急処置ステップを表示する
- ユーザーが症状詳細画面に遷移したとき、システムはNG行為を赤字で表示する

### イベント駆動要件

- ユーザーが赤レベル（即病院）症状を選択したとき、システムは広告を表示しない
- ユーザーがオフライン状態のとき、システムは全症状情報をローカルから取得する
- ユーザーが3症状目を閲覧したとき、システムはペイウォールを表示する

### 状態駆動要件

- アプリがプレミアムプランの間、システムは広告を非表示にする
- アプリがフリープランの間、システムは1日3症状の閲覧制限を適用する

### オプション要件

- ユーザーが位置情報を許可したとき、システムは最寄りの動物病院を表示できる（P1）

### 望ましくないイベント

- 緊急通信エラー時でも、システムはローカルキャッシュから症状情報を提供しなければならない

## Part 3: 画面仕様

### 画面一覧

| 画面 | 概要 | P0/P1 |
|------|------|-------|
| Onboarding | 3ステップ説明（緊急度色分け/オフライン/症状選択） | P0 |
| Home | 犬/猫タブ + 症状カテゴリグリッド | P0 |
| SymptomList | カテゴリ内の症状リスト | P0 |
| SymptomDetail | 緊急度+応急処置ステップ+NG行為 | P0 |
| Paywall | プレミアム訴求 | P0 |
| Settings | プレミアム/プライバシー/通知 | P0 |
| PetProfile | ペット情報入力 | P1 |
| HospitalSearch | 位置情報ベース病院検索 | P1 |

### 画面遷移ルール

```
Onboarding → Home → SymptomList → SymptomDetail
                                      ↓
                                  Paywall (3症状目)
                                      ↓
                                   購入完了 → Home
```

## Part 4: 非機能要件

### パフォーマンス

| 指標 | 目標 |
|------|------|
| 起動時間 | 2秒以内 |
| 症状検索応答 | 0.5秒以内 |
| アプリサイズ | 50MB以下 |

### 信頼性

- オフライン時もP0機能100%動作
- クラッシュ率: 0.5%以下（Crashlytics計測）

### ユーザビリティ

- 緊急時UI: 起動→症状タップ→ガイド表示が3タップ以内
- フォントサイズ可変対応
- Dynamic Type対応

### コンプライアンス

- App Store Review Guideline 5.1.1（医療系）対応
- 「医療診断ではなく情報提供」を明示
- プライバシーポリシー・利用規約をFirebase Hosting配信
- Privacy Manifest対応（Apple 2024）
- GDPR/CCPA対応（Firebase Analytics利用同意）

### 多言語

- 39言語対応（flutter-template標準）
- 症状名・応急処置は専門用語を含むため、医療辞書ベースの翻訳を使用
